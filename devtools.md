## Elements

* Styles - раздел в котором описываются все стили выделенного элемента по классам и селекторам
* Computed - раздкел где эти стили можно структурировать в более удобном виде(списком),
плюс есть возможность семантично структрировать по трем группам: Layout(grid), Text(fonts), Appearance(color, background)

## Console

* Раздел позволяющий просмотреть ошибки выводящиеся в лог, верхнеуровнево можно понять в каком месте произошла ошибка,
имеет смысл использовать для начала анализа и исследования проблемы

## Sources

* Раздел посволяющий максимально эффективно дебажить код, и предоставляющий весь инструментарий, аналогичный средам разработки!
для выполнения дебага
* Поиск и открытие файла через `cmd + P` - macos, `ctrl + P` - window
Панель управления дебагингом:
* F8 - переход к следующему Breakpoint по стеку выполения
* F10 - переход на следующую строку - синхронного выполнения (или на асинхронное выполнение, если до него дошла очередь)
* F11 - проваливается внутрь функции на той строке которая выбрана, функции читаются слева направо
* F12 - выходит из функции на уровень выше
* Watch - раздел позволяющий выяснить состояние лексического окружения текущей функции и ее собственное состояние
* Breakpoints - контроллер для всех точек остоновок
* Scope - раздел позволяющий изучить состояние локального состоятния функции,
всех замыканий более высокого удовня, вплоть до глобального окружения
* Callstack - раздел позволяющий отследить все вызовы до определенного уровня глубины,
по всей цепочке вызовов, таким образом можно понять откуда и куда переходило управление в приложении,
иногда помогает понять природу ошибки или логики

## Network

* Раздел для мониторинга сетевых запросов
* Есть широкий круг возможностей фильтрации и уточнения запросов
* Есть возможность получать различные данные в табличном виде(можно выбирать видимость стобцов)
* Есть возможность просматривать детальную информацию по запросам при выборе
* В детальной информации можно выяснить все подробости(url,headers,types,body,response и т д)

## Performance

* Данный раздел позволяет понять насколько ресурсоемко наше приложение, 
и какую нагрузку оказывает на процессор в тот или иной момент,
позволяет определить наиболее наруженные процессы

## Memory

* Данный раздел показывает текущую потребляемую оперативную память приложения,
так же полезно для выявления наиболее нагруженных страниц

## Application

* Раздел для получение информации о неком глобальном состоянии приложения, которое можно хранить в нескольких разделах:
* Cookies - инфо об аутентификации, токены или любая другая информаци, которую приложение может хранить
* Local storage - локальное браузерное хранилище данных,
аналог базы данных с специальным api хранит json (хранит данные даже вне сессии, но почистить тоже довольно просто)
* Session storage - работает как Local storage, но в рамках сессии
* IndexedDB - браузерная база данных, управляется с помощью специального api
* Cache - описывает все кэшированные ресурсы(у нас таких кстати нет)

## Lighthouse

* Раздел агрегирующий статистику по состоянию приложения, и судя по этому анализатору приложение у нас очень
не эффективно и не производительно

## Recorder

* Раздел позволяющий работать со сценариями, т е в реальном времени задавать некоторые сценарии приложения и повторять их,
* дополняю какими-либо событиями, интервалами или другими операциями (из-за hashRouter для нашего приложения это не сработает)

## Redux

* Раздел для мониторинга состояния приложения в технологии Redux
* Можно посмотреть все состояние приложения в текущий момент, где и какие данные являются актуальными
* Добавлен раздел RTKQ где можно мониторить все query с которым в текущий момент приложение уже работало и какие там данные

## Components

* Раздел похожий на Elements, но необходимый для работы с компонентами реакта, где семантически показано дерево всех копонентов
* Можно все текущие пропсы компонента и в целом проводить разный анализ исходя из этих данных
* Альтернатива для дебагинга, если нужно найти компонент именно в дереве и понять что ему не пришло

## Profiler

* Раздел для мониторинга производительности и времени работы и отклика компонентов реакта
* Полезно для точечного выявления медленных компонетов и их дальнейшей отимизации
